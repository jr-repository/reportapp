DEPLOY TRACE APP KE HOSTINGER
=============================

1. GAMBARAN STRUKTUR HOSTING
----------------------------
Domain/subdomain Anda sudah diarahkan ke:

public_html/traceapp/public/

Itu sudah BENAR untuk CodeIgniter 4.

Berarti struktur upload di hosting harus seperti ini:

public_html/traceapp/
|- app/
|- public/
|- vendor/
|- writable/
|- tests/                (opsional, boleh ikut upload, tapi tidak wajib untuk production)
|- composer.json
|- composer.lock
|- .env
|- spark
|- lainnya

Karena document root sudah mengarah ke folder public, maka:
- public/index.php TIDAK perlu diubah
- app/Config/Paths.php TIDAK perlu diubah
- halaman/page aplikasi TIDAK perlu diubah untuk deployment


2. FILE YANG PERLU DISESUAIKAN
------------------------------
Untuk deploy ke Hostinger, yang perlu Anda ubah/fokuskan hanya file konfigurasi dan permission:

A. .env
   Ini file PALING PENTING.
   Yang harus disesuaikan:
   - app.baseURL
   - app.forceGlobalSecureRequests
   - database.default.hostname
   - database.default.database
   - database.default.username
   - database.default.password
   - database.default.port

B. public/.htaccess
   Normalnya TIDAK perlu diubah karena domain Anda sudah langsung menunjuk ke folder public.
   File ini hanya perlu disentuh kalau:
   - route masih 404
   - mod_rewrite tidak aktif
   - Anda ingin paksa redirect ke HTTPS

C. Permission folder
   Yang harus writable:
   - writable/
   - writable/cache/
   - writable/logs/
   - writable/session/
   - public/Uploads/
   - public/Uploads/Reports/

D. app/Config/App.php
   Tidak wajib diubah kalau .env terbaca dengan benar.
   File ini cukup jadi fallback saja.

4. KONDISI FILE .ENV SAAT INI
-----------------------------
Saat ini file .env project lokal masih berisi konfigurasi database lokal:

database.default.database = reportappdb
database.default.username = root
database.default.password = 020204

Jadi SEBELUM upload ke hosting, Anda harus ganti nilainya ke database hosting.

Jika Anda tidak ingin mengubah .env lokal permanen, ada 2 cara:

Cara 1:
- Upload semua file
- Lalu edit file .env langsung di Hostinger File Manager

Cara 2:
- Buat file .env khusus production di lokal
- Upload file itu ke public_html/traceapp/.env


5. LANGKAH DEPLOY LENGKAP
-------------------------
Langkah 1. Pastikan versi PHP hosting
- Minimal PHP 8.2
- Karena project ini require:
  - php ^8.2
  - CodeIgniter 4.7
  - dompdf
  - firebase/php-jwt

Langkah 2. Pastikan extension PHP aktif
- mysqli
- mbstring
- intl
- json
- openssl
- gd
- fileinfo
- dom

Kalau PDF error atau blank, biasanya masalah ada di:
- dom
- mbstring
- gd

Langkah 3. Aktifkan SSL
- Masuk hPanel Hostinger
- Aktifkan SSL untuk domain/subdomain traceapp.devmagank.site
- Setelah aktif, gunakan https

Langkah 4. Upload file project
- Upload SELURUH isi project ke:
  public_html/traceapp/

Berarti hasil akhirnya folder public harus ada di:
  public_html/traceapp/public/

Jangan upload isi folder public langsung ke public_html/traceapp/ jika document root Anda memang sudah diarahkan ke public_html/traceapp/public/.

Langkah 5. Pastikan file hidden ikut terupload
- File .env itu hidden file
- Kadang File Manager tidak menampilkan/ikut upload dotfile
- Kalau .env sulit diupload:
  - upload file env biasa
  - lalu rename di hosting menjadi .env

Langkah 6. Edit file .env di hosting
- Buka file:
  public_html/traceapp/.env
- Isi dengan konfigurasi production yang sudah ditulis di bagian 3

Langkah 7. Cek document root domain
- Pastikan domain/subdomain traceapp.devmagank.site benar-benar menunjuk ke:
  public_html/traceapp/public/

Kalau ini salah, gejalanya biasanya:
- 404
- folder listing
- index.php tidak jalan
- asset CSS/JS tidak terbaca

Langkah 8. Pastikan .htaccess ikut terupload
- File yang harus ada:
  public_html/traceapp/public/.htaccess

Kalau .htaccess tidak ada, route clean URL bisa gagal.

Langkah 9. Set permission folder
- Set permission minimal:
  - writable = 755 atau 775
  - writable/cache = 755 atau 775
  - writable/logs = 755 atau 775
  - writable/session = 755 atau 775
  - public/Uploads = 755 atau 775
  - public/Uploads/Reports = 755 atau 775

Kalau upload foto gagal, hampir pasti permission public/Uploads/Reports kurang.

Langkah 10. Bersihkan file cache lama bila perlu
- Jika ada masalah cache setelah upload:
  - hapus isi writable/cache/
  - hapus isi writable/session/
  - jangan hapus file index.html jika ada

Langkah 11. Uji aplikasi setelah deploy
Uji minimal:
- buka login
- login user
- buka dashboard
- buat draft laporan
- upload foto
- submit laporan
- buka detail laporan
- generate PDF
- tes PWA install prompt di HP


6. FILE YANG TIDAK PERLU DIUBAH
-------------------------------
Karena struktur hosting Anda sudah tepat, file berikut normalnya tidak perlu diubah:

- public/index.php
- app/Config/Paths.php
- app/Views/*
- app/Controllers/*

Catatan:

- yang paling menentukan tetap .env dan HTTPS


7. OPSIONAL TAPI SANGAT DISARANKAN
----------------------------------
A. Paksa semua request ke HTTPS
Karena PWA butuh HTTPS, sebaiknya final gunakan:

app.baseURL = 'https://traceapp.devmagank.site/'
app.forceGlobalSecureRequests = true

B. Update rule www ke https bila perlu
Di public/.htaccess saat ini ada rule redirect non-www ke http:

RewriteCond %{HTTPS} !=on
RewriteCond %{HTTP_HOST} ^www\.(.+)$ [NC]
RewriteRule ^ http://%1%{REQUEST_URI} [R=301,L]

Rule itu tidak memaksa HTTPS penuh, hanya menghapus www ke http.

Kalau nanti Anda ingin canonical full HTTPS, bisa ganti blok itu menjadi:

RewriteCond %{HTTPS} !=on [OR]
RewriteCond %{HTTP_HOST} ^www\.(.+)$ [NC]
RewriteRule ^ https://traceapp.devmagank.site%{REQUEST_URI} [R=301,L]

Tapi:
- ini OPSIONAL
- lakukan hanya kalau SSL sudah aktif
- kalau belum yakin, tidak usah diubah dulu


8. TROUBLESHOOTING CEPAT
------------------------
Masalah: 500 Internal Server Error
Cek:
- file .env salah
- PHP version < 8.2
- extension PHP kurang
- permission writable salah
- ada file yang tidak ikut terupload

Masalah: database tidak connect
Cek:
- hostname MySQL benar atau tidak
- username/password database benar
- database name benar
- user sudah di-assign ke database di hPanel

Masalah: CSS/JS tidak tampil
Cek:
- baseURL salah
- domain belum mengarah ke folder public
- file Assets tidak ikut terupload

Masalah: route 404
Cek:
- public/.htaccess ada
- Apache rewrite aktif
- document root benar ke public_html/traceapp/public/

Masalah: upload foto gagal
Cek:
- permission public/Uploads/Reports
- ukuran upload max di PHP Hostinger
- post_max_size
- upload_max_filesize

Masalah: PDF gagal generate
Cek:
- extension dom aktif
- mbstring aktif
- gd aktif
- file vendor ikut terupload lengkap

Masalah: PWA/install app tidak muncul
Cek:
- domain harus HTTPS
- service worker harus bisa diakses
- buka dari browser HP yang support
- manifest.json terbaca


9. REKOMENDASI FINAL DEPLOY
---------------------------
Urutan paling aman:

1. Upload semua file project ke public_html/traceapp/
2. Pastikan domain mengarah ke public_html/traceapp/public/
3. Edit .env production
4. Aktifkan SSL
5. Set permission writable dan Uploads
6. Tes login, create report, upload foto, PDF, dan PWA


10. RINGKASAN PALING PENTING
----------------------------

4. Gunakan HTTPS, bukan HTTP
5. Permission writable dan public/Uploads/Reports harus benar
